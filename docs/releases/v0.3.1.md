# AgentForge v0.3.1 — Patch

> Released 2026-06-17. Coordinated release train (ADR-0015): every
> workspace package bumps to `0.3.1` in lockstep.

A small patch train: one correctness fix found by dogfooding the
*published* 0.3.0 wheel, plus a security-relevant dependency refresh.
No API changes — safe to upgrade from 0.3.0.

---

## Highlights

- **`--version` reports the real version again (bug-024).** On a clean
  PyPI install, `agentforge --version` printed `0.0.0+unknown` and every
  package's `__version__` read a stale `0.2.3`. The CLI was looking up the
  wrong distribution name (`agentforge` instead of `agentforge-py`, broken
  since the v0.2.1 rename), and the `__version__` strings were hardcoded
  literals the release bump never touched. Both now source the version from
  installed distribution metadata, so they **can't drift again** — and the
  release process no longer has to touch `__version__` at all.

- **pyjwt security update.** Bundles pyjwt 2.13.0 (five advisories). Two
  matter for the MCP auth seam shipped in 0.3.0 (enh-003): an algorithm
  allow-list bypass with `PyJWK`/`PyJWKClient` (`GHSA-jq35-7prp-9v3f`) and
  `PyJWKClient` accepting `file://`/`data:` URIs → local-file read / SSRF
  (`GHSA-993g-76c3-p5m4`), plus an HS256 algorithm-confusion forge, a
  detached-payload DoS, and a JWKS cache-wipe DoS.

---

## Fixed

- **bug-024 (P3)** — version reporting sourced from distribution metadata
  everywhere (CLI `--version`, every package `__version__`). See
  [bug-024](../bugs/bug-024-version-reporting.md).

## Security

- **pyjwt 2.12.1 → 2.13.0** (five advisories; see above).

## Changed

- Dependency bumps (lockfile): cryptography 48.0.0 → 48.0.1, aiohttp
  3.14.0 → 3.14.1, python-multipart 0.0.28 → 0.0.31, litestar 2.21.1 →
  2.22.0.

## Anti-drift

- The packaged-wheel E2E gate (`scripts/packaged_e2e.py`) now asserts the
  **installed** artifact reports its real version (not the fallback) — the
  packaging-level check the bug-024 source-tree unit test can't provide.
  Runs per-PR and in `release.yml` before publish.

---

## Upgrade

```
pip install -U agentforge-py
```

No code changes required. Every workspace member is `0.3.1`.
